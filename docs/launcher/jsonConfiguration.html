JSON Configuration

The Launcher script allows for the flexible placement of browsers on the Campfire Screen. The JSON files themselves can be located within the StreamingAssets folder within the Unity Project, or you may place them within the directory: C:/[User]/Campfire

Samples​

Fullscreen Browsers
"name":"RPI Website",
"description":"Opens up the RPI home page and the GSAS hub.",
"tag":"Classic",
"browsers":[
{
    "url":"https://www.rpi.edu/",
    "display":"wall"
},
{
    "url":"https://hass.rpi.edu/games-and-simulation-arts-and-sciences",
    "display":"floor"
}]

Demo Fields​

The demo and browser fields are decoded within the JSON reader.

Name, Description, & Tag​

These are all used to fill out the Launcher Item within the Launcher.

Unity Scene​

The core scene of the demo, usually a camera scene (FlatCamera or SectionedCamera). Leaving blank will open up the FlatCamera Scene.

...
    "unityScene":"SectionedCamera",				
...

Additive Scenes​

Any additional scenes that need to be added in to be loaded on top of the core scene. This is a list which can additively load multiple scenes.

...
    "additiveScenes":[{"scene":"CampusTour"}],	
...

Origin & Rotation​

Where the player should be placed once the scene loads. These vectors are based on having the Y-axis as the vertical axis.

...
    "origin":{"x":0,"y":20,"z":0},
    "rotation":{"x":0,"y":90,"z":0},
...

Browsers​

A list of the different browsers that will be opened. These are custom classes as defined below.

...
"browsers":[
{
    "url":"https://www.rpi.edu/",
    "display":"wall"
},
{
    "url":"https://hass.rpi.edu/games-and-simulation-arts-and-sciences",
    "display":"floor"
}]
...

Browser Fields​

URL​

The URL that the browser will open. Can be an address hosted on the internet, or a locally hosted URL using localhost.

Display​

The display that the browser will target. This will accept the following inputs:
[Wall, wall, aux, Aux, auxillary, Auxillary, floor, Floor]

Section​

The section of the display that the browser will center itself on. These start at index 0. The wall by default will have 5 sections and the floor and auxillary displays will have 1 section. Leaving blank will default to the center section of either display (section 2 on the wall, section 0 on the floor and auxillary display.)

Aux Display​

The auxillary display to target, if there are multiple connected.

Sort Order​

The order in which to sort the rendering of overlapping browsers. Higher values will show on top of lower values.

Anchors​

The point of the section from (0,0) to (1,1) to act as the reference for placement, where the center is (0.5, 0.5). Used if you want to align parts of the browser to corners or edges, or anywhere in between. By default, this is the center of the section: (0.5, 0.5).

Pivot​

The point on the browser to act as the center of the object. All placements and rotations are in reference to this pivot. By default this is the center of the browser: (0.5, 0.5).

Position​

The position of the browser relative to its anchor and pivot in pixels. This allows for uncentered placement of the browser.

Size​

The size of the browser relative to its section. A value of (1,1) will have the browser take up the entire section. A value of (5,1) will take up the entire wall display. By default, browsers on the floor and aux displays have a size of (1,1) and browsers on the wall have a size of (5,1).

Moveable​

If this is true, you can move around the browser with a Campfire Select context menu by right clicking on the browser.
